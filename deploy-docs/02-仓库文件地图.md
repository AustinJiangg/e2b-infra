# 02 仓库文件地图

本篇逐一说明仓库里每个文件/目录的作用，以及它们与部署机上 `/opt/e2b-infra/` 的对应关系。
改任何东西之前，先在这里找到它的**源头**——直接改部署机上的文件都是临时的，会被下次
`rpm -Uvh` 或 `build.sh -i` 覆盖（详见 `single-node-offline-deploy.md` §5.0）。

## 1. 仓库根目录

## 2. `e2b-deploy/`：部署包的可读镜像

`e2b-deploy.tar.gz`（Source9）才是 RPM 真正用的输入；`e2b-deploy/` 目录是它的**可读副本**，
专为 code review 存在——每次更新压缩包时同步更新目录，git diff 才能看出改了什么（根 README 有说明）。

两者内容一致，除了目录里**有意省略**：
- `dep/ubuntu-22.04-custom.tar.gz`（约 70MB 的离线模板基础镜像，只在压缩包里）；
- 压缩包内自带的 `.git/`。

> 因此**重建压缩包不能直接 tar 这个目录**（会丢 ubuntu 镜像），正确流程见 03 篇 §6。

### 2.1 顶层

| 文件 | 装到 | 作用 |
|---|---|---|
| `build.sh` | `/opt/e2b-infra/build.sh` | **单机部署总入口**：`-i` 安装组件、`-s` 启动部署、`-d` 停止清理、`-u` 卸载、`-f` 全量重部署、`-r <job>` 快速重跑单 job、`-m <镜像>` 制作沙箱基础镜像。详解见 04/05 篇 |
| `build_prod.py` | `/opt/e2b-infra/build_prod.py` | 构建模板的示例脚本：`python build_prod.py <别名>` 用 `FROM harbor:443/e2b-orchestration/ubuntu:22.04-custom` 建模板（内嵌 token 是示例值，实际以 `/root/.e2b/config.json` 为准） |

### 2.2 `dep/`：离线依赖 + 单机增强版脚本（overlay）

`dep/` 里的脚本与 RPM 从上游源树装出来的**同名文件是两套**：`build.sh -i` 的 `install_e2b`
会把 dep 版**覆盖**到 `/opt/e2b-infra/` 顶层（即 runbook 说的 "dep overlay"）。dep 版包含
单机部署需要的全部增强（幂等化、离线缓存、ACL token 持久化、`--only` 快速重跑等）。

**配置/凭据类**：

| 文件 | 覆盖到 | 作用 |
|---|---|---|
| `.env` | `/opt/e2b-infra/.env`（基线） | **部署总配置**：`SERVER_IP`（装机第一件事就是改它）、所有端口/镜像/桶名/凭据占位。逐项含义见 04 篇 §3.7 |
| `template-manager.hcl` | `/opt/e2b-infra/nomad/template-manager.hcl` | 增强版 job 模板：`ORCHESTRATOR_SERVICES="orchestrator,template-manager"` 合体进程 + `E2B_FC_LAUNCH_MODE`/`MAX_STARTING_INSTANCES_PER_NODE` 两个开关的**持久化源头** |
| `default.hcl` | （参考样例，不自动安装） | 一份生成好的 `/etc/nomad.d/default.hcl` 样例，便于比对 |
| `daemon.json` | `install_docker` 用（当前该函数被注释） | docker 配置样例：insecure-registries + 国内镜像加速 |
| `harbor.cnf` / `nginx.conf` | `/etc/nginx/ssl/harbor.cnf`、`/etc/nginx/nginx.conf` | 自签证书请求配置（CN=harbor，SAN DNS:harbor，CA:TRUE）与 nginx 反代配置（443→127.0.0.1:2900，`client_max_body_size 2G`） |
| `minio.yml` / `minio.service` | `/etc/default/minio.yml`、`/etc/systemd/system/minio.service` | MinIO 环境变量（数据目录、minioadmin 凭据）与 systemd 单元 |

**部署脚本类（dep overlay 的主体）**：

| 文件 | 作用 |
|---|---|
| `deploy.sh` | 渲染 `nomad/*.hcl` → `rendered/` 并提交 job；构建/推送镜像；seed 初始用户；放开 tier 配额。**带 `--only <job>`**（`build.sh -r` 依赖它）。详见 05 篇 §10 |
| `start-server.sh` | server 侧引导：调 `install-consul.sh`/`install-nomad.sh` + `run-consul.sh`/`run-nomad.sh`（server 模式） |
| `start-client.sh` | client 侧引导：业务二进制 → `/usr/bin`，宿主机调优（swap/大页/sysctl/nbd udev/dnsmasq）。单机模式下 consul/nomad client 的启动调用已注释（由 server 那个 agent 兼任 client） |
| `init-client.sh` | 沙箱运行环境铺设：envd/内核/firecracker 三件套到 `/fc-*`，持久化 sysctl，大页 |
| `run-consul.sh` / `run-nomad.sh` | 真正生成配置、写 systemd、启动、**ACL bootstrap 与 token 持久化/恢复**（`/opt/consul/acl.token`、`/data/nomad/acl.token`、回写 `.env`） |
| `install-consul.sh` / `install-nomad.sh` | 安装二进制：优先用 `/tmp` 缓存的 zip（`-i` 提前拷好），装到 `/opt/{consul,nomad}` 并 symlink 到 `/usr/local/bin` |
| `uninstall-nomad.sh` | 停/清 Nomad：purge 全部 job、stop 服务、卸 alloc 挂载、删 `/etc/nomad.d` `/data/nomad` `/log/nomad` 与二进制（默认 dry-run，`--force` 才真删） |

**离线大件**（体积大，**不入 git**，装机前手动放进部署机 `/opt/e2b-infra/dep/`；清单见
`single-node-offline-deploy.md` §1）：`consul_1.21.4_linux_arm64.zip`、`nomad_1.10.4_linux_arm64.zip`、
`harbor-offline-installer-aarch64-v2.13.0.tgz`、`minio`。
（`ubuntu-22.04-custom.tar.gz` 在压缩包内已带。）

**SDK 补丁源文件**（`install_e2b` 拷进 python site-packages，配合 `patch_e2b.py`）：

| 文件 | 拷到 | 作用 |
|---|---|---|
| `connection_config.py` | `e2b/` | SDK 连接配置（http 化/自定义 API 地址） |
| `code_interpreter_sync.py` | `e2b_code_interpreter/` | 同步 run_code 的 http 化 |
| `dockerfile_parser.py` | `e2b/template/` | 模板 Dockerfile 解析适配 |
| `build_api.py`、`main.py` | `e2b/template_sync/` | 模板构建 API 的同步实现适配 |

## 4. 装机后 `/opt/e2b-infra/` 从哪来（对应关系速查）

| 部署机路径 | 来源 |
|---|---|
| `bin/*`（orchestrator、api、envd、client-proxy、seed-db、goose、vmlinux.bin、firecracker、fc-launch、fc-netns-exec、migrations…） | RPM `%build` 编译产物 + Source 二进制（03 篇 §5 有完整清单） |
| `bin/*.Dockerfile` | 上游源树各 `packages/*/Dockerfile`（patch 改造过） |
| `nomad/*.hcl` | 上游 `iac/provider-gcp/nomad/jobs/*.hcl`（patch 改造过）；其中 `template-manager.hcl` 会被 dep overlay 覆盖 |
| 顶层 `deploy.sh`、`start-*.sh`、`run-*.sh`、`install-*.sh`、`uninstall-*.sh`、`init-client.sh` | RPM 先铺**上游 iac 版**，`build.sh -i` 再用 **dep 增强版**覆盖（⚠️ rpm 升级后必须重放 overlay，见 runbook §5.0） |
| `build.sh`、`build_prod.py`、`dep/` | `e2b-deploy.tar.gz` 原样铺入 |
| `patch_e2b.py`、`helm/` | Source7 / 上游源树 |
| `.env`（顶层）、`rendered/`、`harbor/` | **不在 RPM 里**，是 `-i`/`-s`/`deploy.sh` 的运行产物 |
